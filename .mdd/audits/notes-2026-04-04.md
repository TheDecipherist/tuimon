### [02] Static File Server
**Files read:** src/server.ts (163 lines), src/__tests__/server.test.ts (103 lines)
**Findings:**
- HIGH: Path traversal check on line 90 is bypassable. `join(rootDir, safePath)` normalizes `..` but `!filePath.startsWith(rootDir)` fails when rootDir doesn't end with separator (e.g., `/tmp/foo` vs `/tmp/foobar/secret`). Fix: use `path.resolve()` and compare against `rootDir + path.sep`.
- MEDIUM: `readFileSync` in request handler blocks event loop. Acceptable for local dev tool.
- MEDIUM: `decodeURIComponent` on line 72 can throw `URIError` on malformed percent-encoding. No try/catch.
- LOW: No Content-Length header on responses
- LOW: Recursive port scanning (up to 100 deep) — a loop would be safer
- LOW: Test temp dirs never cleaned up
**Test coverage:** 9 tests / ~12 expected paths. Missing: path traversal attempt, malformed URL, binary files
**Doc accuracy:** Matches implementation
